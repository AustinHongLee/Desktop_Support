# 桌面輔助系統 — 面板優化與延伸定位 設計建議

> 版本:v0.1 (2026-05-15)
> 範圍:面板 UI 視覺與互動、穩定性與錯誤處理、程式架構與可維護性、視窗/面板定位邏輯、新外掛 (plugins) 延伸方向
> 目的:在不重寫的前提下,提出可分階段落地的優化清單與優先順序。

---

## 0. 一張圖看現況

```
┌──────────────────── DockWindow (PyQt6, Frameless, StaysOnTop) ─────────────────┐
│ TailBtn │ 標題 │ 指令 │ 最近指令 │ 最近檔案 │ 最近資料夾 │ 來源 │ context_label │
│         │      │      │           │           │             │      │  ... 拉伸  │

                                               │ EdgeMenu      │ Quit
                                               ▼
                                        CommandPalette ──► ActionRunner ──► WorkerHost
                                                                ▼
                                                          JobMonitor (logs)

State: AppStateStore  (edge / screen_name / auto_hide / recent_*)
Plugins: launcher/plugins/* + plugin.json + actions.json (動態載入)
```

整體骨架已經乾淨:**Registry / Runner / WorkerHost / StateStore / Dock** 五層分得清楚,Plugin 也走 JSON 宣告。優化重點不是重寫,而是把幾個目前耦合在 `DockWindow` 裡的職責拉出去、把使用者最常感受到的不順手點補起來。

## 1. 現況評估摘要

### 1.1 優點
- 採宣告式 plugin (`plugin.json` + `actions.json`),新增功能不需要碰 UI。
- `worker_host` 走子行程 + JSON 事件流,UI 不會被 plugin 卡死。
- `AppStateStore` 已經把邊緣 / 自動收合 / 最近紀錄持久化。
- 已支援 dock 四邊吸附 + 自動收合(tail mode)。

### 1.2 主要痛點
| # | 痛點 | 出現位置 |
| - | --- | --- |
| P1 | `DockWindow` 同時負責 UI、版面、定位、context 輪詢、樣式、子視窗管理,單檔 600+ 行 | `launcher/ui/dock_window.py` |
| P2 | QSS 樣式 hardcode 在每個視窗,無主題系統 | `dock_window._apply_style` / `command_palette._apply_style` / `job_monitor._apply_style` |
| P3 | `_snap_to_edge` 在 collapsed / 4 邊 / 多螢幕條件下用 if-else 堆出來,難測試 | `dock_window.py` 428-471 行 |
| P4 | `enterEvent / leaveEvent / mousePressEvent / _collapse_if_idle` 之間的展開/收合狀態機散在 5 個方法,出現過「滑進去後沒展開」的可能 | `dock_window.py` 124-145、522-544 |
| P5 | `CommandPalette` 沒有 fuzzy 比對、沒有鍵盤上下鍵 hint、沒有 group by 分類 | `command_palette.py` |
| P6 | `JobMonitor` 是純文字 log,artifact / error / progress 沒有區隔 | `job_monitor.py` |
| P7 | `ActionRunner` 對 worker 失敗(stderr、非零 return code)沒有顯眼提示,UI 只看到 log 結尾一行 | `runner.py` |

| P9 | 沒有設定面板(preferences),邊緣/螢幕/快捷鍵/外觀只能透過選單或改檔 | 全域缺項 |

## 2. 面板 UI 優化建議

### 2.1 視覺與版面

目標:從「Win95 風格的工程工具列」升級為「現代化 system bar」,但仍保留高密度資訊。

| 項目 | 建議 |
| - | - |
| 主題 | 新增 `launcher/ui/theme.py`,集中放 light / dark / high-contrast 三組色票,所有視窗從一個 `apply_theme(widget, theme)` 統一注入。 |
| 字型 | 標題用 `Segoe UI Variable Display`,內文用 `Segoe UI Variable Text`(回退 `Microsoft JhengHei UI`);中文與英文混排會明顯變整齊。 |
| 圓角與陰影 | dock 視窗保留 frameless 但加 1px outer-stroke + 軟陰影(`QGraphicsDropShadowEffect`),避免融進桌布。 |
| 間距 | 目前 spacing=6、margin=10/4,水平模式按鈕高度 30 — 建議 spacing=8、margin=12/6、按鈕高度 32,避免按鈕間擠在一起。 |
| 圖示 | 改用 `qtawesome` 提供的 Material/Fluent 圖示,取代 Qt standardIcon(`SP_FileDialogDetailedView` 在不同 Windows 版本長相差很多)。 |
| 狀態色 | context label 依來源加色點:Explorer 綠、拖放 藍、最近 灰、手動 橘 — 讓使用者一眼知道資料怎麼來的。 |
| Tail 模式 | 收起來的 tail 條目前是純文字「工具列」/「工\n具」;建議改成「圖示 + 1 個小型 context 指示燈(綠/灰)」,佔位 16px 不變但資訊量更高。 |

### 2.2 互動行為

| 項目 | 建議 |
| - | - |
| 拖動 dock | 加入 `mousePressEvent / mouseMoveEvent` 偵測 alt+左鍵拖動,鬆開後自動 `_snap_to_edge`(吸到最近邊);目前只能用「位置→指定螢幕/邊」選單。 |
| 焦點區隔 | 把「快速指令」按鈕加上 visual primary style(深色填滿),其他改 ghost style,避免使用者每次掃描所有按鈕。 |
| Recent 清單 | 把「最近指令 / 最近檔案 / 最近資料夾」三顆按鈕合成一顆「最近」按鈕 + sub-menu tab,水平模式可省 2 顆按鈕寬度。 |
| 拖放回饋 | `dragEnterEvent` 接收時 dock 邊框變藍、顯示「放開以設為目前 context」提示;目前接受拖放但無視覺回饋。 |
| 鍵盤導覽 | dock focus 時 Tab/Shift+Tab 在按鈕間走;`Esc` 收合到 tail。 |
| 雙擊 tail | 雙擊 tail 切換 auto-hide 開關(目前只能展開)。 |
| 防誤觸 | leaveEvent 收合的 900ms 有時太短(滑鼠移到 menu 時會被取消但 menu 關閉後立刻倒數),建議改 1500ms,且當 `QApplication.activePopupWidget()` 一關閉就 reset timer。 |

### 2.3 Command Palette

`CommandPalette` 是這個系統的靈魂,但目前比較像「過濾框 + 列表」。建議:

1. **Fuzzy + 拼音/首字母**:`thefuzz` 或自己寫 subsequence match,讓「pdf」可以命中「PDF 壓縮」、輸入「fzy」命中「複製檔名」;中文用 pypinyin 取得首字母再加進 haystack。
2. **分組顯示**:依 `action.category` 分群,每群最多顯示 5 個,並支援 `Ctrl+1..9` 直接執行第 N 個。
3. **預覽區**:右側 30% 預覽 action 描述、上次執行紀錄、需要的 context 條件,使用者一眼判斷會不會用錯。
4. **歷史排序**:加權「最近 7 天使用次數」+「字串匹配分數」,讓常用 action 自然浮上來(已經有 `recent_actions`,只是 palette 沒用它)。
5. **Esc 行為**:有輸入字時 Esc 清空,空字串時 Esc 才關閉。
6. **無 match 時引導**:沒結果時顯示「沒有匹配,試試 `/all` 看全部、或 `?` 看說明」。

### 2.4 Job Monitor

目前是單一 `QPlainTextEdit`。建議拆兩列:

```
┌──────────────────────────────────────────────────┐
│ ▶ PDF 壓縮 · running · 00:00:12       [取消][複製]│
├──────────────────────────────────────────────────┤
│ 進度條 (若 plugin 回報 progress 事件)            │
├──────────────────────────────────────────────────┤
│ ▤ Log    | 📎 Artifacts | ⚠ Errors              │
│ -------------------------------------------------│
│ [info]   開始處理 sample.pdf                     │
│ [done]   1.23s                                  │
└──────────────────────────────────────────────────┘
```

技術上:
- `JobEvent.type` 已有 `artifact` / `error` / `progress` 概念 → 在 monitor 中分流到不同 tab。
- 失敗時(`result.return_code != 0`)整顆視窗紅色 highlight,並暴露「打開 log file / 複製錯誤」按鈕。
- 加上「取消」按鈕,把 `ActionRunThread` 換成可控的 `QProcess` 或在 `ActionRunner` 提供 `cancel()`,實際發 SIGTERM 給 worker_host。

## 3. 程式架構與可維護性

### 3.1 拆解 DockWindow

把 `DockWindow` 切成 3 個共生物件:

```
DockWindow (QWidget)            ← 純 UI 容器、layout
 ├─ DockController              ← 收合狀態機 / context 輪詢 / action 執行
 ├─ EdgePositioner              ← 螢幕、edge、collapsed 幾何計算 (純函式可單測)
 └─ DockMenuFactory             ← 建構 recent_files / recent_folders / context / edge 子選單
```

`EdgePositioner` 拆出來最容易立刻見效:
- 輸入:`available_geometry`、`edge`、`collapsed`、`is_vertical_layout`、`screens_info`
- 輸出:`(width, height, x, y, layout_direction)`
- 對 dock 來說,從 `_snap_to_edge / _collapsed_geometry / _apply_edge_layout` 抽出來變一個 dataclass 計算 + 套用兩步,單元測試覆蓋 4 邊 × {展開, 收合} × {主/副螢幕} = 16 種狀態。

### 3.2 樣式集中與主題

新增:

```
launcher/ui/theme.py
  ├─ Theme dataclass (colors, fonts, radius, spacing)
  ├─ DEFAULT_LIGHT / DEFAULT_DARK / HIGH_CONTRAST
  └─ apply_theme(widget, theme)  → 產生 QSS 字串注入

launcher/ui/styles/
  ├─ dock.qss.j2     (Jinja2 模板,變數從 Theme 注入)
  ├─ palette.qss.j2
  └─ monitor.qss.j2
```

好處:換主題只需要替換 Theme 物件,不必去每個視窗的 `_apply_style` 改字串。

### 3.3 設定中心 (Preferences)

目前 `AppStateStore` 同時放「使用者偏好」(edge / auto_hide / screen) 與「歷史紀錄」(recent_*)。建議拆成兩個 JSON 檔:

```
%LOCALAPPDATA%/EngineeringLauncher/
  ├─ settings.json     ← 偏好 (穩定、會被使用者打開來改)
  └─ history.json      ← 最近紀錄 (頻繁寫入、可以隨時清空)
```

並提供:
- `PreferencesDialog`:在 dock 「位置」選單最下方加一個「偏好設定...」,GUI 化目前的選項並加上「主題」「快捷鍵」「自動收合延遲時間」。
- `settings.schema.json`:對應 JSON Schema,寫檔前驗證,避免使用者手改壞掉。

### 3.4 安全寫檔

`state_store._save` 改 atomic write,避免電源中斷後 state.json 變成 0 byte:

```python
def _save(self) -> None:
    tmp = self.path.with_suffix(self.path.suffix + ".tmp")
    tmp.write_text(json.dumps(...), encoding="utf-8")
    os.replace(tmp, self.path)   # atomic on Windows
```

並把 `_load` 的全攔截 `except Exception` 改成具體 `JSONDecodeError / OSError`,且寫入 logger,讓資料壞掉時看得到。

### 3.5 Plugin 生命週期

目前 `ActionRegistry` 只在啟動時 load 一次。建議:

- 加 `registry.reload()` 並掛在「位置」選單下「進階 → 重新載入外掛」。
- 把 `plugin.json` 多支援 `version` / `requires` / `author`,顯示在偏好設定的「外掛清單」頁。
- 在 `actions.json` 加 `enabled: true|false`,讓使用者可以關掉某 plugin 但不刪檔。
- `actions.json` schema 校驗(jsonschema),plugin 寫錯時開機就跳警告而不是 crash。

### 3.6 全域熱鍵

`QShortcut(Ctrl+K)` 只在 dock 視窗 active 時有效。建議用 `keyboard` 或 `pyqthotkey`/`pynput` 註冊系統熱鍵(預設 `Ctrl+Alt+Space` → 喚出 palette、`Ctrl+Alt+D` → 展開/收合 dock),並讓使用者在偏好設定改鍵。

## 4. 穩定性與錯誤處理

| 項目 | 建議 |
| - | - |
| 統一 logger | 在 `launcher/core/logging_config.py` 設好 `%LOCALAPPDATA%/EngineeringLauncher/logs/launcher.log`,RotatingFileHandler 5MB×5;所有 except 都改成 `logger.exception(...)`。 |
| Crash 攔截 | `sys.excepthook` + Qt 的 `qInstallMessageHandler`,出錯時彈一個「複製錯誤 / 開啟 log 目錄」對話框並寫檔。 |
| Worker 退場碼 | `ActionRunner.run` 完成後若 `return_code != 0` 且 events 中沒有 `error` 事件,主動補一條 `JobEvent(type="error", message=f"worker exited with {return_code}")`,避免 JobMonitor 只顯示 `失敗 (1)`。 |
| Worker 卡死 | 加 `runner.run(..., timeout=None)`,可由 action 在 `actions.json` 宣告 `timeout_seconds`;超時 `process.kill()` 並回報 `timeout` 事件。 |
| Single instance | 目前用 `SingleInstanceGuard("Local\\EngineeringLauncher")`,但 `--set-context` 時是寫進 ContextInbox 後直接 return — 第二次啟動的 context 確定會被原本 process 撿到嗎?建議在 inbox 加 `mtime` 檢查 + 主行程定時 poll(已經有,但要驗證寫入時序)。 |
| Self test | `run_self_test` 建議擴充:啟動所有 plugin、跑一次 dry-run action、檢查 state.json 可讀寫;CI / release 前都跑一次。 |
| 單元測試 | 目前只有 `test_context_model.py`,建議加:`test_state_store_atomic`、`test_edge_positioner`、`test_registry_loader`、`test_runner_event_parsing`(各 < 30 行,但能擋住 80% 的回歸)。 |

## 5. 延伸定位 — 視窗/面板定位邏輯

這一段是你問題裡明確點出來的「延伸定位」,把它當成一個獨立子題處理。

### 5.1 現有行為盤點

`AppStateStore` 持久化 `edge`、`screen_name`、`auto_hide_enabled`。`_snap_to_edge` 的策略是:
- 水平(top / bottom):寬度 = 螢幕寬,高度 = 40
- 垂直(left / right):寬度 = 118,高度 = 螢幕高
- 收合:水平 → 132×18,垂直 → 18×132
- 螢幕找不到 → 預設 primary,再不行 → cursor 所在

### 5.2 待補的定位行為

| # | 行為 | 建議 |
| - | - | - |
| L1 | **記憶最後一次的精確位置** | 即使吸邊,也存「相對於該邊的偏移」,讓使用者拖過一次後下次重開仍在同位置;螢幕拔掉時 fallback 到 primary 但保留偏移比例。 |
| L2 | **多螢幕 hot-plug** | 監聽 `QApplication.screenAdded / screenRemoved`,目標螢幕拔掉時自動轉去 primary,避免 dock 消失在已斷開的螢幕上。 |
| L3 | **DPI / scale 變更** | 連 `QScreen.physicalDotsPerInchChanged`,觸發 `_snap_to_edge`;目前 DPI 改了不會重排版。 |
| L4 | **避開 Windows 工作列** | `availableGeometry` 已經會避開大部分情況,但「自動隱藏的工作列」會回傳整片;建議用 `SHAppBarMessage(ABM_GETTASKBARPOS)` 取真實 taskbar 區域作為禁區。 |
| L5 | **避開最大化的應用程式** | 可選功能:偵測前景視窗 maximize 時,自動退讓 4px 透明邊,避免重要 UI 被遮。 |
| L6 | **吸到「指定的螢幕邊角」** | 除了 4 邊外,新增 4 個角(TL/TR/BL/BR)mini-mode:只顯示「指令」+「context 點」,適合不想佔整條邊的使用者。 |
| L7 | **跟隨 context 的彈出定位** | 例如 Explorer context 變更時 dock 短暫亮一下(已有 `_poll_context_inbox` 觸發 show),可進一步在游標所在的螢幕短暫展開 2 秒再回到原螢幕的 tail。 |
| L8 | **熱區感應展開** | tail 模式時,把展開觸發從「點擊 tail」擴成「滑鼠停在螢幕邊緣 ≥150ms」,類似 Windows 自動隱藏工作列。 |
| L9 | **多 dock(進階)** | 同一台多螢幕時,允許每個螢幕都掛一個 dock(共享 state),不必跨螢幕拉滑鼠。 |

### 5.3 EdgePositioner 草擬簽名

```python
# launcher/ui/edge_positioner.py
@dataclass(frozen=True)
class DockPlacement:
    geometry: QRect
    orientation: Literal["horizontal", "vertical"]
    is_collapsed: bool

class EdgePositioner:
    def __init__(self, prefs: DockPrefs, screens: list[QScreen]) -> None: ...
    def compute(self) -> DockPlacement: ...
    def snap_nearest_edge(self, current: QPoint) -> str: ...
    def excluded_rects(self) -> list[QRect]:  # taskbar / monitor 之間
        ...
```

`DockWindow` 改成只負責:
```python
placement = self._positioner.compute()
self.setFixedSize(placement.geometry.size())
self.move(placement.geometry.topLeft())
self._apply_orientation(placement.orientation)
```

這也讓「定位邏輯」可被單元測試 — 不必跑 Qt 主迴圈。

## 6. 延伸定位 — 新外掛 / 應用場景

依現有架構(每個 plugin 一個 `plugin.json` + `actions.json` + python module),以下 plugins 都可以在不動 UI 的前提下加進來。按「ROI / 實作難度」排序。

### 6.1 立刻就能加(< 1 天)

| Plugin | Action 範例 | 為什麼適合 |
| - | - | - |
| `quick_screenshot` | 「截取目前資料夾名稱所在螢幕」「擷取選取檔案的縮圖」 | 利用 `QScreen.grabWindow` + 寫到 context.folder,完成後丟進 JobMonitor 的 artifacts 區 |
| `clipboard_history` | 「貼上前 N 筆剪貼簿文字」「把剪貼簿圖片存到目前資料夾」 | 已有 `windows/clipboard.py`,擴成 ring buffer 即可 |
| `archive_tools` | 「把選取檔案打包成 zip」「解壓縮到此」 | 用 stdlib `zipfile`、`tarfile`,無外部依賴 |
| `hash_tools` | 「計算 SHA256 / MD5」「驗證雜湊比對」 | 工程師高頻使用,接 `hashlib` |
| `text_transform` | 「檔名 → 駝峰 / 蛇形 / kebab」「批次大小寫」 | 補 `rename_tools` 的不足 |

### 6.2 中等(2–5 天,有外部依賴)

| Plugin | Action 範例 | 依賴 |
| - | - | - |
| `ocr_tools` | 「OCR 選取 PDF / 圖片」「截圖後 OCR 並複製」 | `tesseract` + `pytesseract`,或本機 PaddleOCR |
| `ai_summarize` | 「摘要選取 PDF」「翻譯選取檔案」「依檔名生成標籤」 | Anthropic / OpenAI API key 從偏好設定讀;走 worker_host 子行程,key 不入 git |
| `code_quickopen` | 「在 VS Code 開啟資料夾」「以 cwd 開新 Cursor」「PowerShell here」 | 已有 PowerShell here,擴成多 IDE 偏好 |
| `image_pipeline` | 「批次轉 WebP」「批次縮圖 1080px」「去 EXIF」 | `Pillow` |
| `git_quick` | 「在此 git status / pull / 開 origin 網頁」 | `subprocess` + 偵測 `.git` |

### 6.3 長期(視 ROI 決定)

| Plugin | 場景 | 備註 |
| - | - | - |
| `daily_brief` | 早上開機自動展開 dock,顯示昨日整理過的檔案、今日 todo | 與 `daily_brief` skill 概念重疊 |
| `connector_bridge` | 把選取檔案推到 Slack / Notion / Box | 走 MCP-style 介面,獨立 connector layer |
| `workflow_recorder` | 錄製一連串 action → 存成 macro action,變成「一鍵跑完」 | 需要 plugin 之間能互呼;先設計 macro DSL |
| `voice_command` | 按 hotkey 用語音下指令到 palette | 用 whisper.cpp 在本機跑 |

### 6.4 產品定位的兩條延伸路線

1. **「工程師的 Alfred / Raycast for Windows」**
   - 賣點:Command Palette + 從 Explorer 抓 context + 多 plugin。
   - 行銷對象:嵌入式 / FAE / 機構工程師(原本目標族群)、Windows 上找不到 Raycast 替代品的人。
   - 競品:PowerToys Run、Flow Launcher、Wox。差異化:context-aware(從 Explorer / drop / 剪貼簿三路抓),plugin 是 declarative JSON。

## 8. 不建議現在做的事(避免擴散)

- **重寫成 QML / Flutter / Electron**:目前 PyQt6 + 子行程 worker 已經夠用,改 stack 風險遠大於收益。
- **加 plugin 市集 / 遠端載入**:plugin 走檔案系統就好,線上市集會帶入簽章、權限模型等大坑。
- **把 plugin 改成 Python class import**:現在子行程 + JSON 事件流就是最大防線,plugin crash 不會殺主程序;改 in-process 會喪失這個保護。
- **跨平台 (macOS / Linux)**:`windows/explorer_context.py`、`clipboard.py`、`SHAppBarMessage` 全都是 Win32 專有,要跨平台等於拆掉一半;先把 Windows 體驗做到 9 分再說。
